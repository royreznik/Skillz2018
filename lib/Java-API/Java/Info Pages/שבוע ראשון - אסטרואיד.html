<h1>שלב האתגרים - שבוע 1</h1> <p>השבוע, נוסיף למשחק את אובייקט האסטרואיד.</p> <ul> <li> <p>אסטרואיד הוא אובייקט עגול במשחק. לעומת רוב האובייקטים במפה, האסטרואיד אינו נקודתי, אלא בעל רדיוס, ולכן כל האינטראקציות איתו נעשות תוך התחשבות ברדיוס הזה.</p> </li> <li> <p>פיראטים יכולים לדחוף אסטרואידים, והשימוש בפונקציית canPush הוא בדיוק אותו הדבר עבור האסטרואידים.</p> </li> <li>שימו לב: מכיוון שלאסטרואיד יש גודל, מספיק שחלק ממנו יהיה ברדיוס הדחיפה של הפיראט על מנת שיהיה אפשר לדחוף אותו.</li> <li>אסטרואיד שנדחף, ינוע בכיוון שהוא נדחף בו עד שהוא ידחף שוב על ידי פיראט אחר או עד שהוא יצא מהמפה.</li> <li>אסטרואיד שיצא מחוץ למפה ימות, ויחזור למשחק בנקודה ההתחלתית שלו אחרי מספר מסוים של תורות.</li> <li>אם אסטרואיד פוגע בפיראט, הפיראט מתרסק.</li> <li>אם שני אסטרואידים מתנגשים, שניהם מתרסקים.</li> </ul> <p><br></p> <h2>שינויים ב-API</h2> <ul> <li>שינויים ב-PirateGame:<ul> <li>הפעולה ()getAllAsteroids - מחזירה את רשימת כל האסטרואידים במשחק (בין אם הם נמצאים על המפה ובין אם לא).</li> <li>הפעולה ()getLivingAsteroids - מחזירה את רשימת כל האסטרואידים שנמצאים כרגע במפה. שימו לב - היא יכולה להיות ריקה!</li> </ul> </li> <li>המחלקה SpaceObject: למשחק נוספה המחלקה SpaceObject, ממנה יורשים Pirate ו- Asteroid. לכל האובייקטים מסוג זה קיים המאפיין הבא:<ul> <li>size - שמציין את הגודל של האובייקט (עבור פיראטים, הגודל הוא 0).</li> </ul> </li> <li>למחלקה Asteroid יש תכונות שימושיות רבות, כגון:<ul> <li>maxSpeed - המהירות של האסטרואיד</li> <li>spawnTurns - מספר התורות עד שהאסטרואיד חוזר לחיים</li> <li>direction - הכיוון אליו ילך האסטרואיד, אם לא ידחף במהלך התור הנוכחי</li> </ul>
